# 🗺️ Roadmap Belajar Zig untuk Pythonista

> *"Kamu sudah tahu cara memasak dengan microwave (Python). Sekarang kita belajar masak dengan api langsung — lebih ribet, tapi hasilnya lebih powerful."*

## SILABUS LENGKAP

### FASE 1 — The Syntax Shock *(Minggu 1-2)*
> *Tujuan: Bisa membaca dan menulis kode Zig tanpa panik*

| Bab | Topik | Konsep Kunci |
|-----|-------|--------------|
| 1.1 | Instalasi & Hello World | `zig init`, `zig build run` |
| 1.2 | Kenapa ada `;` dan `{}` | Statement vs Expression |
| 1.3 | Static Typing — Tipe Data Wajib Ditulis | `i32`, `u8`, `f64`, `bool` |
| 1.4 | Variabel: `const` vs `var` | Immutability by default |
| 1.5 | Fungsi & Return Type | `fn namaFungsi(param: Tipe) TipeReturn` |
| 1.6 | Control Flow | `if`, `while`, `for`, `switch` |
| 1.7 | Error Handling ala Zig | `!`, `try`, `catch` — bukan `try/except` |
| 🏋️ | **Latihan**: Kalkulator CLI sederhana | |

### 👶 FASE 2 — The Manual Power *(Minggu 3-5)*
> *Tujuan: Paham memory, pointer, dan kenapa Zig tidak butuh GC*

| Bab | Topik | Konsep Kunci |
|-----|-------|--------------|
| 2.1 | RAM & Memory — Analogi Loker Sekolah | Stack vs Heap |
| 2.2 | Pointer — Alamat Rumah, Bukan Rumahnya | `*T`, `&variable` |
| 2.3 | Array vs Slice — Python List vs Zig | `[5]i32` vs `[]i32` |
| 2.4 | String di Zig — Kenapa Ribet? | `[]const u8`, `[]u8` |
| 2.5 | Allocator — Siapa yang Bayar Sewa RAM? | `std.heap.GeneralPurposeAllocator` |
| 2.6 | Struct — Mirip Class Python, tapi Beda | `struct`, method, `self` |
| 2.7 | Enum & Tagged Union | Lebih powerful dari Python Enum |
| 🏋️ | **Latihan**: Linked list sederhana dari scratch | |

### 🧑 FASE 3 — The Comptime Magic *(Minggu 6-8)*
> *Tujuan: Menulis kode generic dan meta-programming ala Zig*

| Bab | Topik | Konsep Kunci |
|-----|-------|--------------|
| 3.1 | Apa itu Comptime? | Kode yang jalan saat *kompilasi*, bukan runtime |
| 3.2 | `comptime` vs Python Decorators | Perbandingan langsung |
| 3.3 | Generic Function dengan Comptime | `fn foo(comptime T: type)` |
| 3.4 | Generic Struct | `Stack(comptime T: type)` |
| 3.5 | Reflection — Inspeksi Tipe saat Kompilasi | `@TypeOf`, `@typeInfo` |
| 3.6 | Build System Zig | `build.zig` — ganti Makefile/CMake |
| 3.7 | Testing di Zig | `test "nama" {}`, `zig test` |
| 🏋️ | **Latihan**: Generic `Stack(T)` dengan push/pop/peek | |

### 🧙 FASE 4 — Building Systems *(Minggu 9-12)*
> *Tujuan: Bikin sesuatu yang nyata dan bisa dipakai Python*

| Bab | Topik | Konsep Kunci |
|-----|-------|--------------|
| 4.1 | I/O & File System | Baca/tulis file, stdin/stdout |
| 4.2 | CLI App Profesional | Argument parsing, exit codes |
| 4.3 | Interop dengan C Library | `@cImport`, linking |
| 4.4 | Membuat Shared Library (`.so`/`.dll`) | Dipanggil dari Python via `ctypes` |
| 4.5 | Python ↔ Zig Bridge | `cffi` / `ctypes` integration |
| 4.6 | Optimasi & Benchmark | `ReleaseFast`, profiling |
| 4.7 | Concurrency Dasar | `async`/`await` di Zig |
| 🏋️ | **Proyek Akhir**: Library Zig yang bisa di-`import` di Python | |

## PERSIAPAN SEBELUM MULAI

**Install Zig:**
```bash
# Pakai zigup (recommended)
curl https://zigup.zig.guide/install | sh

# Atau download langsung dari ziglang.org
# Versi stabil: 0.13.0
zig version  # Verifikasi instalasi
```

**Setup Editor (VSCode):**
```
Extension: "Zig Language" by ziglang
Extension: "ZLS" (Zig Language Server)
```

**Setup ala Gh0sT4n**
- [Halaman 1](./1.md)

**Mental Model sebelum mulai:**
| Python | Zig |
|--------|-----|
| Interpreted | Compiled |
| Garbage Collected | Manual Memory |
| Duck Typing | Static Typing |
| Slow tapi nyaman | Cepat tapi tanggung jawab besar |
| Runtime errors | Compile-time errors (lebih aman!) |
